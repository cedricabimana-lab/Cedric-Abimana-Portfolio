# Cedric Abimana — Portfolio Site

This folder contains your full portfolio site, ready to upload to GitHub and publish as a free, shareable link via GitHub Pages.

## Files

- `index.html` — homepage (About, Experience, Focus Areas, Achievements, Education, Testimonials, Contact)
- `projects.html` — projects overview page
- `project-meta.html` — Meta project detail page (interactive mockup)
- `project-midea.html` — Midea project detail page (interactive mockup)
- `project-headway.html` — Headway project detail page (interactive mockup)
- `project-goodtime.html` — GoodTime candidate portal project detail page (interactive mockup)
- `cedric-headshot.png` — used elsewhere on the site
- `Cedric_Abimana_Resume_2025.pdf` — downloadable resume, linked from the nav and hero buttons

All files must stay in the same folder for the links between pages (and the resume download) to work.

## How to publish with GitHub Pages

1. Go to [github.com](https://github.com) and create a new repository (for example, `cedric-portfolio`). Keep it public.
2. On the repository page, click **Add file → Upload files**, then drag in every file from this folder (all the `.html` files, the `.png`, and the `.pdf`).
3. Scroll down and click **Commit changes**.
4. In the repository, go to **Settings → Pages** (in the left sidebar).
5. Under **Build and deployment**, set **Source** to **Deploy from a branch**, choose the `main` branch and the `/ (root)` folder, then click **Save**.
6. GitHub will give you a live URL shortly after, usually in this format:
   `https://your-username.github.io/cedric-portfolio/`
7. That link is what you share with recruiters. It updates automatically any time you upload new files to the repository.

No build tools, installs, or command line needed. Once it's uploaded and Pages is turned on, GitHub hosts it for free.
